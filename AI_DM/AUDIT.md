## 🚨 Critical Failures (P0 Blockers)

### 1. `rom_integrator` Skill Crashes on Load
**Impact:** The entire ROM integration pipeline is dead.  
**Error:** `cannot use 'dict' as a set element (unhashable type: 'dict')`  
**Root Cause:** `zones.json` stores `source_roms` as a list of **dict objects** (full ROM metadata), but `rom_integrator.py` tries to add them directly into a Python `set()`, which requires hashable strings.

**Broken code** (`engine/skills/rom_integrator.py`, lines ~109-110):
```python
for p in z.get("source_roms", []):
    self.integrated_paths.add(p)   # p is a dict, not a string!
```

**Result:** `rom_integrator run`, `report`, `zones_list`, and `audit` commands are unavailable. Only 139 commands load instead of 155+.

### 3. AI Mod Generator Produces FAKE `.zobj` Files
**File:** `ai_full_mod_generator.py`  
**Claim:** "Exports N64-compatible `.zobj` files using Fast64"  
**Reality:** It writes 65-byte **text placeholders**.

Example `actor_5609.zobj`:
```
N64 Object: darklink
ID: actor_5609
Generated by AI Dungeon Master
```

This is **not a valid N64 binary object**. The Fast64 export code is commented out:
```python
# Fast64 export commands would go here
# For now, create a placeholder export file
```

The `blender_output/` directory contains Python scripts and JSON scenes, but **no actual 3D models**.
